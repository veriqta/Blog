## Project 1: Highly Available Web Application on AWS

**What you're building:** A real production-grade web application that survives data center failures, scales automatically, and follows AWS Well-Architected best practices.

**Architecture Overview:**
- **VPC** with public and private subnets across **2 Availability Zones**
- **Application Load Balancer (ALB)** in public subnets
- **Auto Scaling Group** of EC2 instances in private subnets
- **NAT Gateway** for outbound internet from private instances
- **Bastion Host** for secure administrative access
- **Security Groups** with least-privilege access

**Why this matters in interviews:**
> *"I designed and deployed a highly available web application across multiple availability zones with auto-scaling, load balancing, and secure private subnet architecture - achieving 99.99% availability."*

### Why This Architecture?

| Component | Purpose | Production Reality |
|-----------|---------|-------------------|
| **VPC** | Isolated network for your resources | Every production app lives in a VPC |
| **Public Subnets** | Resources that need direct internet access (ALB, NAT GW, Bastion) | ALB faces the internet; everything else hides |
| **Private Subnets** | Resources that should NEVER be directly accessible from the internet (EC2 apps) | Security best practice — your app servers are invisible |
| **Multi-AZ** | If one data center fails, the other keeps running | AWS SLA for single AZ = 99.5%. Multi-AZ = 99.99% |
| **ALB** | Distributes traffic across healthy instances | Handles millions of requests, health checks, SSL termination |
| **Auto Scaling** | Automatically adds/removes EC2 instances based on demand | Saves money at night, handles traffic spikes |
| **NAT Gateway** | Allows private instances to reach the internet (for updates, patches) without exposing them | Your servers can `yum update` but hackers can't reach them |
| **Bastion Host** | Secure jump box for SSH access to private instances | "How do you access servers in private subnets?" — Every interview question |
