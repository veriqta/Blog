   # **Part 9: Project 3 — Update Linux, Install Apache, Deploy HTML, and Test**

Welcome back. You're logged into your EC2 instance. Now we'll make it a real web server: update the system, install Apache, configure it, create a website, and test it live. Every command is shown exactly as you type it.

---

## **9.1 Update the Linux System**

Before installing anything, update the package lists and installed packages. This ensures you have the latest security patches.

**Step 1:** Make sure you're at the EC2 prompt:
```
[ec2-user@ip-172-31-xx-xx ~]$
```

**Step 2:** Update the package repository:
```bash
sudo dnf update -y
```

**What this does:**
- `sudo` = run as administrator (superuser)
- `dnf` = package manager (replaces `yum` in Amazon Linux 2023)
- `update` = update all installed packages
- `-y` = automatically answer "yes" to all prompts

**What you should see:**
- Lines scrolling showing packages being checked
- "Complete!" at the end
- The prompt returns

**If you see errors:** Check your internet connection. EC2 instances need outbound internet access (which the default VPC provides). If it fails, wait a moment and try again.

---

## **9.2 Install Apache Web Server**

Apache (`httpd` in Amazon Linux) is the software that serves web pages.

**Step 1:** Install Apache:
```bash
sudo dnf install httpd -y
```

**What this does:**
- `install` = download and install a package
- `httpd` = the Apache HTTP Server package name in Amazon Linux
- `-y` = auto-confirm

**What you should see:**
- "Installing: httpd" and dependencies
- Progress bars or percentages
- "Complete!" at the end

**Step 2:** Verify Apache is installed:
```bash
httpd -v
```

**Expected output:**
```
Server version: Apache/2.4.xx (Amazon Linux)
Server built:   [date]
```

---

## **9.3 Start and Enable Apache**

Installing doesn't automatically start the service. You need to start it now AND configure it to start automatically on boot.

**Step 1:** Start Apache:
```bash
sudo systemctl start httpd
```

**What this does:** Starts the Apache web server process immediately.

**Step 2:** Enable Apache to start on boot:
```bash
sudo systemctl enable httpd
```

**What this does:** Configures Linux to automatically start Apache every time the server reboots.

**Step 3:** Check Apache status:
```bash
sudo systemctl status httpd
```

**What you should see:**
```
● httpd.service - The Apache HTTP Server
     Loaded: loaded (/usr/lib/systemd/system/httpd.service; enabled; preset: disabled)
     Active: active (running) since [date]
```

**Key indicators:**
- `enabled` = will start on boot ✓
- `active (running)` = currently running ✓

**Step 4:** Press `q` to exit the status view and return to the command prompt.

## **9.4 Verify Apache Default Page**

Before deploying your own content, verify Apache is working with its default page.

**Step 1:** Open your local web browser (on your computer, not in the terminal).

**Step 2:** In the address bar, type your EC2 **Public IPv4 address**:
```
http://3.85.123.45
```
(Replace with your actual IP)

**Step 3:** Press Enter.

**What you should see:**
- The **Apache Test Page** — a red and white page saying "It works!"
- The title says "Amazon Linux Test Page"
- This confirms Apache is running and your security group allows HTTP traffic

**If you see "This site can't be reached" or timeout:**
- Check that your instance is running (EC2 console > Instances)
- Check that the security group allows HTTP (port 80) from 0.0.0.0/0
- Check that you used `http://` (not `https://`)
- Wait 30 seconds and try again

## **9.5 Create Your Website Content**

Now we'll replace the default Apache page with your own website. Apache serves files from `/var/www/html/` by default.

### **Navigate to the Web Directory**

**Step 1:** In your SSH terminal, change to the web directory:
```bash
cd /var/www/html/
```

**Step 2:** Verify you're in the right place:
```bash
pwd
```
**Expected output:** `/var/www/html`

**Step 3:** List current files:
```bash
ls -la
```
**Expected output:** Shows `index.html` (the default Apache test page)

---

## **9.6 Set Correct File Permissions**

Apache runs as a special user (`apache`). The web files need to be readable by everyone.

**Step 1:** Verify the file was created:
```bash
ls -la
```

**Expected output:**
```
-rw-r--r-- 1 root root [size] [date] index.html
```

**Step 2:** Set ownership to the Apache user:
```bash
sudo chown apache:apache index.html
```

**Step 3:** Set permissions so Apache can read it:
```bash
sudo chmod 644 index.html
```

**Step 4:** Verify:
```bash
ls -la
```

**Expected output:**
```
-rw-r--r-- 1 apache apache [size] [date] index.html
```

## **9.9 Create a Custom Error Page**

Let's add a custom 404 error page, just like we did with S3.

**Step 1:** Create the error page:
```bash
sudo nano /var/www/html/404.html
```

**Step 2:** Paste this HTML:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>404 - Page Not Found</title>
    <style>
        body {
            font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
            background: #0f172a;
            color: white;
            min-height: 100vh;
            display: flex;
            justify-content: center;
            align-items: center;
            text-align: center;
            padding: 20px;
        }
        .container { max-width: 500px; }
        h1 { font-size: 6rem; margin-bottom: 10px; color: #ef4444; }
        h2 { font-size: 1.8rem; margin-bottom: 20px; }
        p { color: #94a3b8; margin-bottom: 30px; line-height: 1.6; }
        a {
            color: #60a5fa;
            text-decoration: none;
            font-weight: 600;
            border: 2px solid #60a5fa;
            padding: 12px 30px;
            border-radius: 25px;
            transition: all 0.3s ease;
        }
        a:hover {
            background: #60a5fa;
            color: #0f172a;
        }
    </style>
</head>
<body>
    <div class="container">
        <h1>404</h1>
        <h2>Page Not Found</h2>
        <p>The page you're looking for doesn't exist on this server.<br>This is a custom error page running on Apache via Amazon EC2.</p>
        <a href="/">Go Back Home</a>
    </div>
</body>
</html>
```

**Step 3:** Save and exit:
- `Ctrl+O`, `Enter`, `Ctrl+X`

**Step 4:** Set ownership and permissions:
```bash
sudo chown apache:apache /var/www/html/404.html
sudo chmod 644 /var/www/html/404.html
```

**Step 5:** Configure Apache to use this error page. Edit the Apache config:
```bash
sudo nano /etc/httpd/conf/httpd.conf
```

**Step 6:** Scroll down (use Down arrow or Page Down) until you find a section with `<Directory "/var/www/html">`

**Step 7:** Inside that section (after `AllowOverride None` and before `</Directory>`), add this line:
```
ErrorDocument 404 /404.html
```

It should look like:
```
<Directory "/var/www/html">
    AllowOverride None
    ErrorDocument 404 /404.html
</Directory>
```

**Step 8:** Save and exit:
- `Ctrl+O`, `Enter`, `Ctrl+X`

**Step 9:** Test the configuration syntax:
```bash
sudo httpd -t
```

**Expected output:** `Syntax OK`

**Step 10:** Restart Apache:
```bash
sudo systemctl restart httpd
```

**Step 11:** Test the error page. In your browser, type a non-existent URL:
```
http://3.85.123.45/this-page-does-not-exist
```

**You should see:** The custom 404 page with red "404" and "Go Back Home" link.

## **9.10 Troubleshooting Common Issues**

**Problem: "Permission denied" when connecting via SSH**
- **Fix:** Check that your `.pem` file permissions are `400` (`chmod 400 veriqta-ec2-key.pem`)
- Check that you're using the correct username: `ec2-user` (not `root` or `admin`)
- Check that you're using the correct IP address

**Problem: "Connection timed out"**
- **Fix:** Verify the instance is running (EC2 console)
- Verify the security group allows SSH (port 22) from 0.0.0.0/0
- Verify you have a public IP assigned

**Problem: Apache test page shows, but not your custom page**
- **Fix:** Hard refresh browser (Ctrl+Shift+R)
- Check file is in `/var/www/html/index.html`
- Check file ownership is `apache:apache`
- Check Apache is running: `sudo systemctl status httpd`

**Problem: "Forbidden" or 403 error**
- **Fix:** Check file permissions are `644` or `755`
- Check SELinux isn't blocking: `sudo setenforce 0` (temporary) or configure properly

**Problem: Changes not reflecting after editing**
- **Fix:** Always run `sudo systemctl restart httpd` after config changes
- Clear browser cache
