## **8.2 Project 3 Overview — EC2 Web Server**

**What you're building:** A real Linux virtual machine (EC2 instance) running the Apache web server, serving a custom HTML page that you can access from any browser on the internet.

**The Architecture:**

```
[Your Browser]
      ↓
[EC2 Public IP] → [Security Group: Port 80 open]
      ↓
[Apache Web Server] → [HTML Files]
      ↓
[Amazon Linux 2023]
```

**What you'll do:**
- Launch an EC2 instance (virtual machine)
- Configure Security Groups (firewall rules)
- Connect to EC2 via SSH (terminal)
- Update the Linux system
- Install Apache web server
- Configure Apache
- Deploy HTML content
- Test from your browser
- Troubleshoot common issues
- Clean up resources

**Why this matters:** EC2 is the foundation of cloud computing. Understanding how to launch, secure, connect to, and configure a virtual machine is essential for any cloud role. This project teaches you Linux basics, web server administration, and network security — all through hands-on practice.

## **8.3 Launch EC2 — Step by Step**

**Step 1:** Sign in to AWS Console as your IAM user.

**Step 2:** In the search bar, type: `EC2`
- Click on **"EC2"** under Services.

**Step 3:** You'll land on the EC2 Dashboard. Look for the orange button **"Launch instance"** (it's in the "Resources" section or at the top right). Click it.

**Step 4:** The "Launch instance" wizard opens. It has several steps shown on the left sidebar: Name and tags, Application and OS Images, Instance type, Key pair, Network settings, Configure storage, Advanced details, Summary.

---

### **Step 1: Name and tags**

**Instance name:** Type:
```
veriqta-web-server
```

**Tags (optional but recommended):**
- Click **"Add tag"**
- Key: `Project`
- Value: `VERIQTA-Project-3-EC2`

---

### **Step 2: Application and OS Images (Amazon Machine Image)**

This is the operating system for your virtual machine.

**Step 5:** Under "Application and OS Images", you'll see "Quick Start" tab is selected.

**Step 6:** Find and click on **"Amazon Linux"** (it should be the first option in Quick Start).

**Step 7:** Under "Amazon Machine Image (AMI)", select:
- **"Amazon Linux 2023 AMI"** (NOT Amazon Linux 2 — AL2023 is the latest)
- It should say "Free tier eligible" next to it

**Why Amazon Linux 2023?** It's the latest AWS-optimized Linux distribution, free tier eligible, and uses `dnf` as the package manager (modern replacement for `yum`).

---

### **Step 3: Instance type**

This determines the CPU, memory, and performance of your VM.

**Step 8:** Under "Instance type", click the dropdown.

**Step 9:** Select **"t2.micro"** (or **"t3.micro"** if t2.micro is not available)
- It should say "Free tier eligible" next to it
- This gives you 1 vCPU and 1 GB RAM — enough for a web server

**If you see t3.micro instead of t2.micro:** Both are free tier eligible. t3 is newer. Either works.

---

### **Step 4: Key pair (login)**

This creates the security key you'll use to connect to your instance via SSH.

**Step 10:** Under "Key pair name", click the dropdown.

**Step 11:** Select **"Create new key pair"**

**Step 12:** A dialog box "Create key pair" appears:

**Key pair name:** Type:
```
veriqta-ec2-key
```

**Key pair type:** Select **"RSA"**

**Private key file format:**
- **Windows users:** Select **".pem"** (you'll use it with PuTTY or OpenSSH)
- **Mac/Linux users:** Select **".pem"**

**Step 13:** Click **"Create key pair"**

**Step 14:** Your browser downloads a file named `veriqta-ec2-key.pem`
- **CRITICAL:** Save this file to:
  ```
  Documents/VERIQTA-AWS-Projects/Project-3-EC2-WebServer/
  ```
- **Do NOT lose this file.** Without it, you cannot connect to your EC2 instance. AWS does not store it; if you lose it, you must terminate the instance and create a new one.

---

### **Step 5: Network settings**

This controls how your instance connects to the internet and who can access it.

**Step 15:** Under "Network settings", click **"Edit"** (if the settings are collapsed)

**Step 16:** "VPC - required" — Leave as the default VPC (it should say "vpc-xxxxxxxx (default)")

**Step 17:** "Subnet" — Leave as **"No preference"** or select any available subnet

**Step 18:** "Auto-assign public IP" — Select **"Enable"**
- This gives your instance a public IP address so you can access it from the internet

**Step 19:** "Firewall (security groups)" — Select **"Create security group"**

**Step 20:** "Security group name" — Type:
```
veriqta-web-sg
```

**Step 21:** "Description" — Type:
```
Security group for VERIQTA web server - allows SSH and HTTP
```

**Step 22:** "Inbound security groups rules" — These control what traffic is allowed IN to your instance.

You should see one rule already:
- **Type:** SSH
- **Protocol:** TCP
- **Port range:** 22
- **Source type:** Anywhere
- **Source:** 0.0.0.0/0

**This allows SSH from anywhere.** For learning, this is fine. In production, you'd restrict this to your IP address only.

**Step 23:** We need to add a rule for HTTP (web traffic). Click **"Add security group rule"**

**Step 24:** Configure the new rule:
- **Type:** Select **"HTTP"** from the dropdown
- **Protocol:** TCP (auto-filled)
- **Port range:** 80 (auto-filled)
- **Source type:** Select **"Anywhere"**
- **Source:** 0.0.0.0/0 (auto-filled)
- **Description:** Type: `Allow web traffic`

**What this does:** Port 22 (SSH) lets you connect to manage the server. Port 80 (HTTP) lets visitors view your website.

---

### **Step 6: Configure storage**

**Step 25:** Under "Configure storage":
- **Storage:** Leave as **"8"** GB (this is the free tier limit)
- **Volume type:** Leave as **"gp3"** (general purpose SSD)
- **Encrypted:** Leave as **"Not encrypted"** (for learning; in production, encrypt it)

---

### **Step 7: Advanced details**

You can skip this for now. Leave everything default.

---

### **Summary and Launch**

**Step 26:** On the right side, you'll see a "Summary" panel showing:
- Number of instances: 1
- Software: Amazon Linux 2023
- Instance type: t2.micro (or t3.micro)
- Key pair: veriqta-ec2-key
- Security groups: veriqta-web-sg

**Step 27:** Click the orange button **"Launch instance"**

**Step 28:** You'll see a success page: "Successfully initiated launch of instance"
- Click **"View all instances"** (or the "Instances" link in the left sidebar)

## **8.5 Configure Security Groups (Verify)**

Let's double-check the security group rules.

**Step 1:** On your instance details page, click the **"Security"** tab.

**Step 2:** Look at "Inbound rules". You should see:
| Type | Protocol | Port range | Source | Description |
|------|----------|------------|--------|-------------|
| SSH | TCP | 22 | 0.0.0.0/0 | - |
| HTTP | TCP | 80 | 0.0.0.0/0 | Allow web traffic |

**If HTTP (port 80) is missing:**
- Click on the security group name (starts with `sg-`)
- Click **"Edit inbound rules"**
- Click **"Add rule"**
- Type: HTTP, Source: Anywhere

## **8.6 Connect to EC2 via SSH**

Now you'll connect to your Linux server using the terminal. The process differs slightly by operating system.

### **For Mac and Linux Users**

**Step 1:** Open your Terminal application.

**Step 2:** Navigate to the folder where you saved your key pair:
```bash
cd Documents/VERIQTA-AWS-Projects/Project-3-EC2-WebServer/
```

**Step 3:** Set the correct permissions on your key file (SSH requires this):
```bash
chmod 400 veriqta-ec2-key.pem
```

**What this does:** Makes the file readable only by you. SSH will refuse to use it if permissions are too open.

**Step 4:** Connect to your EC2 instance using SSH:
```bash
ssh -i veriqta-ec2-key.pem ec2-user@YOUR-PUBLIC-IP
```

Replace `YOUR-PUBLIC-IP` with your actual public IP address. For example:
```bash
ssh -i veriqta-ec2-key.pem ec2-user@3.85.123.45
```

**Step 5:** The first time you connect, you'll see a message like:
```
The authenticity of host '3.85.123.45 (3.85.123.45)' can't be established.
ECDSA key fingerprint is SHA256:xxxxxxxxxxxxxxxx.
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

**Step 6:** Type exactly: `yes`
- Press Enter

**Step 7:** You should see the Amazon Linux welcome message:
```
   ,     #_
   ~\_  ####_        Amazon Linux 2023
  ~~  \_#####\
  ~~     \###|
  ~~       \#/ ___   https://aws.amazon.com/linux/amazon-linux-2023
   ~~       V~' '->
    ~~~         /
      ~~._.   _/
         _/ _/
       _/m/'
[ec2-user@ip-172-31-xx-xx ~]$
```

**The `[ec2-user@ip-172-31-xx-xx ~]$` prompt means you're logged in.** You're now inside your Linux server.

---

### **For Windows Users**

Windows doesn't have SSH built-in the same way. You have two options:

**Option A — Windows Terminal / PowerShell (Windows 10/11):**
Modern Windows versions include OpenSSH. The steps are identical to Mac/Linux above.

**Option B — PuTTY (Traditional method):**

**Step 1:** Download PuTTY from `https://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html`
- Download `putty.exe` (64-bit or 32-bit based on your system)

**Step 2:** You need to convert your `.pem` key to `.ppk` format:
- Download PuTTYgen from the same page
- Open PuTTYgen
- Click **"Load"**
- Navigate to your `veriqta-ec2-key.pem` file
- Click **"Save private key"**
- Click **"Yes"** when asked about saving without a passphrase
- Save as `veriqta-ec2-key.ppk` in the same folder

**Step 3:** Open PuTTY

**Step 4:** In the "Host Name" box, type:
```
ec2-user@YOUR-PUBLIC-IP
```
Example: `ec2-user@3.85.123.45`

**Step 5:** In the left sidebar, navigate to: **Connection > SSH > Auth > Credentials**

**Step 6:** Under "Private key file for authentication", click **"Browse"**
- Select your `veriqta-ec2-key.ppk` file

**Step 7:** Click **"Open"** at the bottom

**Step 8:** On first connection, click **"Accept"** when warned about the host key

**Step 9:** You should see the Amazon Linux welcome message and the prompt:
```
[ec2-user@ip-172-31-xx-xx ~]$
```

## **8.7 First Commands on Your Linux Server**

You're now connected. Let's run some basic commands to understand your environment.

**Type each command exactly as shown, then press Enter.**

### **Check who you are:**
```bash
whoami
```
**Expected output:** `ec2-user`

### **Check your current directory:**
```bash
pwd
```
**Expected output:** `/home/ec2-user`

### **List files in current directory:**
```bash
ls
```
**Expected output:** (nothing, or empty — the home directory is empty)

### **List files with details:**
```bash
ls -la
```
**Expected output:** Shows hidden files like `.bashrc`, `.bash_profile`, `.ssh/`

### **Check Linux version:**
```bash
cat /etc/os-release
```
**Expected output:** Shows "Amazon Linux 2023" and version information

### **Check available disk space:**
```bash
df -h
```
**Expected output:** Shows filesystem usage. Look for `/` (root) — it should show about 8GB total.

### **Check memory:**
```bash
free -h
```
**Expected output:** Shows total memory (about 957Mi or ~1GB for t2.micro)
