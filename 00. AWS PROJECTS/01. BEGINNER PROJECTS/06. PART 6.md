   # **Part 6: Project 2 — Deploying the Resume to S3, Connecting CloudFront, and Testing**

Welcome back. Your resume is built and tested locally. Now we'll deploy it to AWS — creating a new S3 bucket, uploading your files, configuring access, setting up CloudFront, and testing the live result. Every click, every setting, every verification step is shown below.

---

## **6.1 Create the S3 Bucket for Your Resume**

**Step 1:** Sign in to the AWS Console as your IAM user.

**Step 2:** In the top search bar, type: `S3`
- Click on **"S3"**

**Step 3:** Click the orange button **"Create bucket"**

**Step 4:** On the "Create bucket" page:

**General configuration:**
- **Bucket name:** Use a globally unique name. Format:
  ```
  veriqta-resume-YOURNAME-2026
  ```
  Example: `veriqta-resume-johnsmith-2026`
- **AWS Region:** Select **US East (N. Virginia) us-east-1** (or the same region you used in Project 1)

**Object Ownership:**
- Select **"ACLs disabled (recommended)"**
- Leave **"Bucket owner enforced"** selected

**Block Public Access settings:**
- **Uncheck** "Block all public access"
- All four sub-checkboxes should become unchecked
- Check the acknowledgment box: **"I acknowledge that the current settings might result in this bucket and the objects within becoming public."**

**Bucket Versioning:**
- Leave as **"Disable"**

**Tags (optional):**
- Click **"Add tag"**
- Key: `Project`
- Value: `VERIQTA-Project-2-Resume`

**Default encryption:**
- Leave as **"Server-side encryption with Amazon S3 managed keys (SSE-S3)"**

**Step 5:** Scroll to the bottom. Click **"Create bucket"**

**Verification:** Green success banner: "Successfully created bucket 'veriqta-resume-YOURNAME-2026'"

---

## **6.2 Enable Static Website Hosting**

**Step 1:** In the S3 Buckets list, click on your new bucket name.

**Step 2:** Click the **"Properties"** tab.

**Step 3:** Scroll to the bottom. Find **"Static website hosting"**

**Step 4:** Click **"Edit"**

**Step 5:** Configure:
- Select **"Enable"**
- Hosting type: **"Host a static website"**
- Index document: `index.html`
- Error document: `index.html` (for a resume, we'll route errors back to the main page — or you can create a separate `error.html` if you prefer)
- Click **"Save changes"**

**Step 6:** Scroll back down to "Static website hosting"
- **Copy the "Bucket website endpoint" URL**
- Save it in a text file labeled "Resume S3 Endpoint"
- It looks like: `http://veriqta-resume-YOURNAME-2026.s3-website-us-east-1.amazonaws.com`

---

## **6.3 Upload the Resume Files**

**Step 1:** Click the **"Objects"** tab (you should already be on this tab after saving properties).

**Step 2:** Click the orange button **"Upload"**

**Step 3:** Upload your files. You have two methods:

**Method A — Drag and Drop:**
- Open your file manager showing `Project-2-Cloud-Resume/`
- Select `index.html`
- Also open the `css/` folder and select `style.css`
- Drag BOTH files into the gray upload box on AWS
- You should see two files listed: `index.html` and `css/style.css`

**Method B — Click Add Files:**
- Click **"Add files"**
- Navigate to `Project-2-Cloud-Resume/`
- Select `index.html`
- Click **"Add files"** again
- Navigate to `Project-2-Cloud-Resume/css/`
- Select `style.css`
- Both files appear in the list

**Step 4:** Under "Destination", verify:
- Bucket: `veriqta-resume-YOURNAME-2026`
- The files will be uploaded to their correct paths (`index.html` in root, `style.css` in `css/`)

**Step 5:** Under "Permissions":
- Select **"Grant public-read access"** from the "Predefined ACLs" dropdown
- Check the box: **"I understand the risk of granting public-read access to the specified objects."**

**Step 6:** Click **"Upload"**

**Step 7:** Wait for both files to show "Succeeded"
- Click **"Close"**

**Verification:** In the Objects tab, you should see:
- `css/` (folder) — click it to verify `style.css` is inside
- `index.html` (file)

---

## **6.4 Configure the Bucket Policy**

**Step 1:** Click the **"Permissions"** tab.

**Step 2:** Scroll down to **"Bucket Policy"**

**Step 3:** Click **"Edit"**

**Step 4:** In the Policy text box, paste the following JSON. **Replace `YOUR-BUCKET-NAME`** with your actual bucket name.

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*"
        }
    ]
}
```

Example with actual bucket name:
```json
"Resource": "arn:aws:s3:::veriqta-resume-johnsmith-2026/*"
```

**Step 5:** Click **"Save changes"**

**Step 6:** Confirm any warning about public access.

## **6.5 Test the S3 Resume Website**

**Step 1:** Go to the **"Properties"** tab.

**Step 2:** Scroll to "Static website hosting"

**Step 3:** Click the **"Bucket website endpoint"** link

**What you should see:**
- Your professional resume loads
- Blue gradient header with your name
- All sections: Summary, Skills, Projects, Education, Certifications
- Skills in three columns
- Project cards with technology tags
- Dark footer at the bottom

**If styles don't load:**
- Check that `css/style.css` uploaded to the `css/` folder
- Check your browser's developer console (F12) for 404 errors
- Verify the HTML references `css/style.css` (not just `style.css`)

**If the page shows "Access Denied":**
- Verify the bucket policy is saved correctly
- Verify "Block all public access" is turned OFF
- Verify files were uploaded with public-read access

## **6.6 Create the CloudFront Distribution**

**Step 1:** In the AWS search bar, type: `CloudFront`
- Click **"CloudFront"**

**Step 2:** Click **"Create distribution"**

**Step 3:** Configure the Origin:

**Origin domain:**
- Do NOT select from the dropdown
- Type your S3 static website endpoint manually:
  ```
  veriqta-resume-YOURNAME-2026.s3-website-us-east-1.amazonaws.com
  ```

**Origin path:** Leave blank

**Name:** Leave the auto-filled value or change to `resume-s3-origin`

---

**Step 4:** Default cache behavior:

**Path pattern:** `Default (*)` (leave as is)

**Viewer protocol policy:** Select **"Redirect HTTP to HTTPS"**

**Allowed HTTP methods:** Leave as **"GET, HEAD"**

**Cache key and origin requests:** Select **"Cache policy and origin request policy (recommended)"**

**Cache policy:** Select **"Managed-CachingOptimized"**

**Origin request policy:** Select **"Managed-CORS-S3Origin"**

**Response headers policy:** Select **"Managed-SimpleCORS"**

**Compress objects automatically:** Check this box

---

**Step 5:** Settings:

**Price class:** Select **"Use only North America and Europe"**

**AWS WAF web ACL:** **"None"**

**Alternate domain name (CNAME):** Leave blank

**Custom SSL certificate:** Leave as **"Default CloudFront certificate (*.cloudfront.net)"**

**Supported HTTP versions:** Leave default

**Default root object:** Type exactly: `index.html`

**Logging:** Leave as **"Off"**

**IPv6:** Leave as **"On"**

---

**Step 6:** Scroll to the bottom. Click **"Create distribution"**

**Step 7:** You'll see the Distributions list. Your new distribution shows **"In Progress"**

**Step 8:** Wait for status to change to **"Deployed"**
- This takes 5-15 minutes
- Click the refresh button (circular arrow) periodically

**Step 9:** Once "Deployed", click on your **Distribution ID** (random string like `E123ABC4DEF5GH`)

**Step 10:** Copy the **"Distribution domain name"**
- It looks like: `d1234abcd5ef6.cloudfront.net`
- Save this as "Resume CloudFront URL"

---

## **6.7 Configure Error Handling**

**Step 1:** On your distribution details page, click the **"Error pages"** tab.

**Step 2:** Click **"Create custom error response"**

**Step 3:** Configure:
- **HTTP error code:** Select `404: Not Found`
- **Customize error response:** Select `Yes`
- **Response page path:** Type: `/index.html` (we'll redirect 404s back to the resume)
- **HTTP response code:** Select `200: OK` (soft redirect — shows the resume instead of an error)
- Click **"Create custom error response"**

**Why redirect to index.html?** For a single-page resume, there's only one page. Any wrong URL should just show the resume.

## **6.8 Test the CloudFront Resume**

**Step 1:** Open a new browser tab.

**Step 2:** Type your CloudFront domain with HTTPS:
```

```
(Use your actual distribution domain)

**Step 3:** Press Enter.

**What you should see:**
- Your professional resume loads
- URL starts with `https://`
- All styling is correct
- Layout is responsive

**Step 4:** Test on mobile (or resize your browser):
- The layout should adapt
- Skills should stack in one column
- Header should shrink

**Step 5:** Test error handling:
```
https://d1234abcd5ef6.cloudfront.net/nonexistent-page
```
- Should show your resume (not an error) because of the custom error response

**Step 6:** Test HTTP to HTTPS redirect:
```
http://d1234abcd5ef6.cloudfront.net
```
- Should automatically redirect to `https://`
