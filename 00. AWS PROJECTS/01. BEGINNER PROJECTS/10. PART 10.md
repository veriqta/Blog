## **10.3 Project 3 Cleanup — Terminate EC2 Instance**

AWS charges for running EC2 instances by the hour. Even a stopped instance can incur charges for storage. The only way to fully stop charges is to **terminate** the instance (which permanently deletes it).

### **Step 1: Stop Apache (Optional but Clean)**

In your SSH terminal:
```bash
sudo systemctl stop httpd
```

This gracefully stops the web server before termination.

### **Step 2: Exit the SSH Session**

```bash
exit
```

**Expected output:**
```
logout
Connection to 3.85.123.45 closed.
```

You're back at your local terminal prompt.

### **Step 3: Terminate the EC2 Instance**

**In the AWS Console:**

**Step 1:** Search bar → type: `EC2` → Click **"EC2"**

**Step 2:** In the left sidebar, click **"Instances"** (under Instances)

**Step 3:** Find your instance `veriqta-web-server`
- It should show **"Instance state: Running"**

**Step 4:** Click the **radio button** to the left of your instance

**Step 5:** Click the **"Instance state"** dropdown button at the top right
- Select **"Terminate instance"**

**Step 6:** A confirmation dialog appears: "Terminate instances?"
- It warns: "Terminated instances will remain visible for a short time, then be permanently deleted."
- Click **"Terminate"**

**Step 7:** The instance state changes to **"Shutting-down"**
- Wait 30-60 seconds, then click the refresh button
- It changes to **"Terminated"**

**Verification:** The instance still appears in the list but with state "Terminated". After a few minutes, it disappears completely. Terminated instances stop incurring charges immediately.

## **10.4 Delete the Security Group**

Security groups don't cost money, but cleaning them up is good practice.

**Step 1:** In the EC2 left sidebar, click **"Security Groups"** (under Network & Security)

**Step 2:** Find your security group `veriqta-web-sg`
- If you don't see it, you may need to wait until the instance is fully terminated (security groups attached to running instances can't be deleted)

**Step 3:** Click the **radio button** next to `veriqta-web-sg`

**Step 4:** Click **"Actions"** dropdown at the top right
- Select **"Delete security groups"**

**Step 5:** Confirm deletion. Click **"Delete"**

**If you get an error saying it's in use:** Wait a few more minutes for the instance to fully terminate, then try again.

## **10.5 Delete the Key Pair (Optional)**

AWS doesn't charge for key pairs, but if you want to clean up completely:

**Step 2:** Find `veriqta-ec2-key`
