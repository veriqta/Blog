## **5.1 Project 1 Cleanup**

Cleaning up properly is a critical skill. AWS charges for resources that exist, even if you're not using them. Let's remove everything we created for Project 1.

### **Delete the CloudFront Distribution**

**Step 1:** In the AWS Console search bar, type: `CloudFront`
- Click on **"CloudFront"**

**Step 2:** On the Distributions page, find your distribution.
- Click the **radio button** to the left of your distribution (NOT the distribution ID link).

**Step 3:** Click the **"Disable"** button at the top right.
- A confirmation dialog appears asking "Are you sure you want to disable this distribution?"
- Click **"Disable"**

**Step 4:** Wait for the distribution status to change from "Deployed" to **"Disabled"**
- This takes a few minutes. Refresh the page periodically.

**Step 5:** Once the status shows "Disabled", click the radio button next to your distribution again.

**Step 6:** Click the **"Delete"** button at the top right.
- A confirmation dialog appears asking "Are you sure you want to delete this distribution?"
- Type `delete` in the confirmation box (or click **"Delete"** if no typing is required — AWS changes this interface occasionally)
- Click **"Delete"**

**Step 7:** The distribution disappears from the list. CloudFront distribution deleted.

---

### **Delete the S3 Bucket and All Files**

**Step 1:** In the AWS Console search bar, type: `S3`
- Click on **"S3"**

**Step 2:** Find your bucket: `veriqta-static-website-YOURNAME-2026`
- Click the **radio button** to the left of the bucket name.

**Step 3:** Click the **"Empty"** button at the top right.
- A confirmation dialog appears: "Empty bucket configuration"
- In the text box, type exactly: `permanently delete`
- Click **"Empty"**

**Step 4:** AWS deletes all objects in the bucket. Wait for the green success message: "Successfully emptied bucket."

**Step 5:** Click the **"Exit"** button on the success page, or navigate back to S3 Buckets.

**Step 6:** Find your bucket again. Click the radio button next to it.

**Step 7:** Click the **"Delete"** button at the top right.
- A confirmation dialog appears: "Delete bucket"
- In the text box, type your exact bucket name: `veriqta-static-website-YOURNAME-2026`
- Click **"Delete bucket"**

**Step 8:** The bucket disappears from the list. S3 bucket deleted.

**Verification:** Refresh the S3 Buckets page. Your bucket should not appear. If it does, repeat the empty and delete steps.

## **5.2 Project 2 Overview — The Cloud Resume Challenge**

The Cloud Resume Challenge is a well-known project in the cloud computing community. The original challenge (created by Forrest Brazeal) asks you to build a resume website and deploy it using cloud services. Our version focuses on the core skills: building a professional resume site, deploying it to S3, serving it through CloudFront, and following a complete project workflow.

**What you'll build:** A professional, responsive resume website that showcases your skills, experience, and education. It will be hosted on S3 and delivered via CloudFront — just like Project 1, but with more depth in planning, design, and validation.

**Why this matters:** This isn't a toy project. A deployed resume website is a real portfolio piece. You can share the CloudFront URL with employers. It demonstrates that you understand AWS fundamentals, web development basics, and deployment workflows.
