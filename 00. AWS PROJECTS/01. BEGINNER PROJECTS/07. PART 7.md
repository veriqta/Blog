## **7.1 Add a Simulated Visitor Counter**

A real visitor counter requires a database and API (which is beyond our current scope), but we can simulate one with JavaScript that stores a count in the browser's local storage. This demonstrates the concept and adds interactivity to your resume.

### **Create the JavaScript File**

**Step 1:** In VS Code, click **File > New File**

**Step 2:** Save it as `script.js` inside the `js/` folder:
- Click **File > Save As**
- Navigate to `Project-2-Cloud-Resume/js/`
- File name: `script.js`
- Save as type: "All Files"
- Click **Save**

**Step 3:** Copy and paste the following JavaScript:

```javascript
// VERIQTA Cloud Resume - Visitor Counter Simulation
// This simulates a visitor counter using browser local storage
// In a full Cloud Resume Challenge, this would connect to a database via API

document.addEventListener('DOMContentLoaded', function() {
    // Check if we have a stored count
    let visitCount = localStorage.getItem('veriqtaResumeVisits');
    
    if (visitCount === null) {
        // First visit (for this browser)
        visitCount = 1;
    } else {
        // Increment visit count
        visitCount = parseInt(visitCount) + 1;
    }
    
    // Save the updated count
    localStorage.setItem('veriqtaResumeVisits', visitCount);
    
    // Display the counter if the element exists
    const counterElement = document.getElementById('visitor-counter');
    if (counterElement) {
        counterElement.textContent = visitCount.toLocaleString();
    }
    
    // Add a subtle animation to the counter
    if (counterElement) {
        counterElement.style.opacity = '0';
        setTimeout(() => {
            counterElement.style.transition = 'opacity 0.5s ease';
            counterElement.style.opacity = '1';
        }, 100);
    }
    
    // Log to console for debugging
    console.log('VERIQTA Resume - Visit count (this browser):', visitCount);
});
```

**Step 4:** Save the file (Ctrl+S / Cmd+S).

## **7.2 Improve Accessibility**

Accessibility (a11y) ensures your resume can be read by screen readers and used by people with disabilities. These are small changes with big impact.

### **Update index.html with Accessibility Improvements**

**Step 1:** Open `index.html`.

**Step 2:** Find the `<html>` tag. Change it to include the language:
```html
<html lang="en">
```
(You probably already have this — verify it's there.)

**Step 3:** Add a "Skip to main content" link for keyboard users. Find the `<body>` tag. Immediately AFTER it, add:

```html
<body>
    <a href="#main-content" class="skip-link">Skip to main content</a>
```

**Step 4:** Find the `<main>` tag. Add an `id` to it:
```html
    <main id="main-content">
```

**Step 5:** Add `aria-label` attributes to navigation links. In the contact-info div, update the links:

```html
                <div class="contact-info">
                    <a href="mailto:[your.email@example.com]" class="contact-link" aria-label="Send email to [Your Name]">[your.email@example.com]</a>
                    <span class="separator">|</span>
                    <a href="https://linkedin.com/in/[yourprofile]" target="_blank" class="contact-link" aria-label="Visit LinkedIn profile (opens in new tab)">LinkedIn</a>
                    <span class="separator">|</span>
                    <a href="https://github.com/[yourusername]" target="_blank" class="contact-link" aria-label="Visit GitHub profile (opens in new tab)">GitHub</a>
                </div>
```

**Step 6:** Add `role` attributes to main sections. Update these tags:

Find:
```html
        <header class="header">
```
Change to:
```html
        <header class="header" role="banner">
```

Find:
```html
        <footer class="footer">
```
Change to:
```html
        <footer class="footer" role="contentinfo">
```

**Step 7:** Save `index.html`.

## **7.3 Add Print-Friendly Styles**

When someone prints your resume (or saves as PDF), it should look professional on paper — no background colors that waste ink, proper page breaks, and readable text.

### **Add Print Media Query to style.css**

**Step 1:** Open `css/style.css`.

**Step 2:** Scroll to the very bottom, AFTER the existing `@media (max-width: 768px)` query.

**Step 3:** Add this new media query:

```css
/* Print Styles */
@media print {
    body {
        background: white;
        padding: 0;
    }
    
    .resume-container {
        box-shadow: none;
        border-radius: 0;
        max-width: 100%;
    }
    
    .header {
        background: #1e293b !important;
        color: white !important;
        padding: 30px;
        -webkit-print-color-adjust: exact;
        print-color-adjust: exact;
    }
    
    .section {
        padding: 20px;
        border-bottom: 1px solid #ddd;
    }
    
    .section-title {
        border-bottom: 2px solid #1e293b;
        color: #1e293b;
    }
    
    .project-item {
        border-left: 3px solid #1e293b;
        break-inside: avoid;
    }
    
    .skill-category h3 {
        color: #1e293b;
    }
    
    .tech-tag {
        background: #e2e8f0;
        color: #1e293b;
        border: 1px solid #cbd5e1;
    }
    
    .footer {
        background: white;
        color: #64748b;
        border-top: 2px solid #e2e8f0;
    }
    
    .counter-box {
        display: none;
    }
    
    a {
        text-decoration: none;
        color: #1e293b;
    }
    
    a[href]:after {
        content: " (" attr(href) ")";
        font-size: 0.8rem;
        color: #64748b;
    }
    
    .contact-link[href]:after {
        content: "";
    }
}
```

**Step 4:** Save `style.css`.

## **7.6 Invalidate CloudFront Cache**

CloudFront caches your files. When you upload new versions, CloudFront might still show the old ones. You need to invalidate the cache.

**Step 1:** Go to AWS Console → CloudFront.

**Step 2:** Click on your distribution ID.

**Step 3:** Click the **"Invalidations"** tab.

**Step 4:** Click **"Create invalidation"**

**Step 5:** In the text box, type:
```
/*
```
This invalidates ALL files in the distribution.

**Step 6:** Click **"Create invalidation"**

**Step 7:** Wait for status to show **"Completed"**
- This takes 2-5 minutes
- Click the refresh button periodically
