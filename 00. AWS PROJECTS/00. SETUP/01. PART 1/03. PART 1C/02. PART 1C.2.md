# Part 1C.2: Understanding the AWS Management Console

## Goal

The goal of this lesson is to help you understand and navigate the AWS Management Console safely.

The AWS Management Console is the browser-based application that provides centralized access to individual AWS service consoles. Its unified navigation can be used to search for services, view notifications, open CloudShell, access account and billing information, and change console settings.

In this lesson, you will learn how to:

* Recognize the main areas of the console.
* Use the search bar.
* Open the Services menu.
* Understand service categories.
* check and change the selected AWS Region.
* Open the account menu.
* Locate billing information.
* Locate security settings.
* Understand the difference between account-wide and Regional information.
* Avoid common beginner mistakes.
* Verify that you can navigate the console confidently.

You will not launch an EC2 instance, create an S3 bucket, or configure CloudFront in this lesson.

The root user is still being used temporarily for initial account setup. It must not become the identity used for everyday AWS work. AWS strongly recommends that the root user be accessed only when a task specifically requires root credentials.

---

# 1. What Is the AWS Management Console?

The AWS Management Console is a graphical interface for interacting with AWS.

It allows you to manage AWS through a web browser instead of entering every instruction through a command-line interface or software development kit.

Through the console, you can:

* Search for AWS services.
* Create cloud resources.
* Review existing resources.
* Change resource settings.
* View monitoring information.
* Review account information.
* Manage security settings.
* Access billing and cost tools.
* Open AWS documentation and support resources.
* Switch between AWS Regions.
* Sign out of the account.

The main console landing page is called **AWS Console Home**.

---

## 1.1 The Console Is a Collection of Service Consoles

AWS does not have one single page containing every possible setting.

Instead, the AWS Management Console provides access to individual service consoles.

Examples include:

```text
AWS Management Console
├── Amazon S3 console
├── Amazon EC2 console
├── IAM console
├── CloudFront console
├── CloudWatch console
├── Billing and Cost Management console
└── Amazon VPC console
```

Each service console has its own:

* Dashboard
* Navigation pane
* Resource pages
* Configuration screens
* Actions
* Search and filter tools

For example:

* The Amazon S3 console manages buckets and objects.
* The Amazon EC2 console manages virtual servers and related resources.
* The IAM console manages identities, roles, policies, and permissions.
* The Billing console displays charges, budgets, payment details, and cost information.

You will move between these service consoles throughout the projects.

---

## 1.2 The Console Is Not the AWS Service Itself

The console is one method of interacting with AWS services.

You can also interact with AWS through:

* AWS Command Line Interface
* AWS software development kits
* AWS APIs
* Infrastructure as Code tools
* Automation platforms

For example, the following two actions can request similar information:

### Console method

1. Open the Amazon S3 console.
2. View the list of buckets.

### AWS CLI method

```bash
aws s3 ls
```

The console is easier for beginners because it makes services and settings visible.

The command line becomes useful when you need:

* Repeatability
* Automation
* Scripting
* Faster administration
* Infrastructure workflows

# 2. Signing In Before Exploring the Console

For this lesson, sign in using the root-user email address and password created during account registration.

This root session is temporary.

## Steps

1. Open the AWS sign-in page.
2. Choose **Root user**.
3. Enter the email address associated with the AWS account.
4. Select **Next**.
5. Enter the root password.
6. Complete any additional verification requested by AWS.
7. Wait for AWS Console Home to load.

The root user is the identity created with the AWS account email address and password and initially has complete access to the account.

---

## Important Root-Session Warning

While exploring the console as the root user:

* Do not create access keys.
* Do not launch resources.
* Do not share your screen publicly.
* Do not publish screenshots containing account or billing information.
* Do not leave the session open on a shared computer.
* Do not save the root password in the browser of a shared device.
* Do not use this session for normal project work.

The root user will be secured and then replaced with an IAM administrative identity for routine learning tasks.

# 4. The Top Navigation Bar

The top navigation bar remains available as you move between many service consoles.

AWS refers to its shared console navigation as **Unified Navigation**. It can provide access to services, search, CloudShell, notifications, support resources, settings, Regions, and account information.

A simplified representation is:

```text
┌─────────────────────────────────────────────────────────────────────┐
│ AWS | Services | Search | CloudShell | Notifications | Region | Account │
└─────────────────────────────────────────────────────────────────────┘
```

The exact arrangement may vary by screen size and console updates.

---

## 4.1 AWS Logo or Console Home Link

The AWS logo or Console Home link normally returns you to the main AWS Console Home page.

Use it when:

* You want to leave a service console.
* You want to return to the dashboard.
* You want to access recently visited services.
* You want a familiar starting point.

Returning to Console Home does not stop or delete any resources.

It only changes the page you are viewing.

* The integrated terminal in Visual Studio Code

# 10. Understanding the Region Selector

The Region selector is one of the most important controls in the AWS console.

Many AWS resources exist in a specific Region.

AWS states that the selected Region determines where many resources are managed. Some services, including IAM, do not require a Region selection.

---

## 10.1 Locate the Current Region

Look near the upper-right area of the navigation bar.

You may see a Region name such as:

```text
N. Virginia
```

or:

```text
US East (N. Virginia)
```

The Region code for that Region is:

```text
us-east-1
```

Other examples include:

```text
US West (Oregon)       us-west-2
Europe (London)        eu-west-2
Europe (Frankfurt)     eu-central-1
Africa (Cape Town)     af-south-1
```

---

## 10.2 Change the Region

To switch Regions:

1. Locate the Region selector in the navigation bar.
2. Select the currently displayed Region.
3. Review the list.
4. Select the required Region.
5. Wait for the service console to reload.

AWS documents this process as selecting the currently displayed Region from the navigation bar and choosing another Region.

---

## 10.3 Select One Main Training Region

Use the same main Region for the Regional resources in this volume unless a lesson specifically states otherwise.

Record it:

```text
My main training Region

Region name: __________________________

Region code: __________________________
```

Before creating an EC2 instance, verify this Region.

Before searching for an EC2 instance, verify this Region again.

---

## 10.4 Regional Resource Example

Suppose you launch an EC2 instance in:

```text
Europe (London)
eu-west-2
```

When the console is displaying:

```text
US East (N. Virginia)
us-east-1
```

the London EC2 instance will not appear in the N. Virginia EC2 resource list.

The resource has not disappeared.

You are looking in a different Region.

---

## 10.5 Changing Region Does Not Move Resources

Selecting another Region does not transfer resources.

Example:

```text
Before switch:

eu-west-2
└── EC2 instance: web-server-01

After console switches to us-east-1:

us-east-1
└── No EC2 instances
```

The original instance remains in `eu-west-2`.

---

## 10.6 Some Services Are Global

IAM does not require the same Region selection used by EC2. CloudFront also operates as a global service even though its origins can be Regional.

You may see wording such as:

```text
Global
```

or the Region selector may not affect the current service page in the same way.

Do not assume a Region has been selected incorrectly simply because a global service behaves differently.

---

## 10.7 Region Selection and S3

S3 buckets are created in a selected Region, although parts of the S3 console can provide an account-wide view of buckets.

Always inspect the bucket’s Region before:

* Uploading files
* Configuring CloudFront
* Troubleshooting endpoints
* Deleting resources

---

## 10.8 Region Selection and Billing

Billing information is account-wide rather than limited to only the Region currently selected in the console.

Changing the Region selector does not hide valid charges from the billing system.

However, cost reports can often be filtered or grouped by Region.

---

## 10.9 Default Region

The console can allow you to set a default Region through Unified Settings.

AWS documents the path as:

1. Open the settings control.
2. Choose to view all user settings.
3. Edit localization and default Region.
4. Select the required default Region.

Setting a default Region can reduce mistakes, but it does not remove the need to check the Region selector.

# 13. Finding the AWS Account ID

The AWS Account ID is a unique 12-digit identifier.

Example:

```text
123456789012
```

You may find it in the account menu or on the account and security credentials pages, depending on the identity and console interface. AWS documents viewing account identifiers from the upper-right account or identity menu and the Security credentials page.

---

## 13.1 Account ID Is Not a Password

The Account ID identifies the account, but it is not enough by itself to sign in or manage resources.

Do not confuse it with:

* Root password
* IAM password
* Access key ID
* Secret access key
* MFA code
* EC2 private key

---

## 13.2 Record It Safely

You may record the Account ID in a private account-administration note.

Do not include it unnecessarily in:

* Public screenshots
* Social media posts
* Website pages
* Public repositories

Although an Account ID is not treated like a password, exposing unnecessary account details can still provide useful information to attackers.

Record:

```text
AWS Account ID: ______________________
```

Never record the root password beside it in plain text.

## 14.7 AWS Budgets

AWS Budgets allows you to define spending or usage thresholds and configure notifications.

* Actual cost exceeds a threshold.
* Forecasted cost may exceed a threshold.
* Usage reaches a configured level.

## 14.8 Cost Anomaly Detection

* Pricing review

## 14.9 Billing Permissions for IAM Users

Creating an IAM administrator does not always make every billing page available automatically.

This setting will be handled deliberately during the IAM administrator lesson.

Do not assume billing access is configured merely because an IAM user has broad administrative permissions.

# 17. Console Actions and Their Meanings

AWS service consoles often use common action labels.

---

## Create

Creates a new resource.

Examples:

* Create bucket
* Create user
* Create distribution
* Create policy

Creation can lead to charges depending on the service.

---

## Launch

Starts a computing resource or deployment workflow.

Example:

```text
Launch instance
```

This can create billable EC2 resources.

---

## Edit

Changes an existing configuration.

Editing security or networking settings can make a resource:

* Accessible
* Inaccessible
* Public
* Private
* Broken
* Less secure

---

## Delete

Removes a resource or configuration.

Deletion may be permanent.

Some resources cannot be deleted until dependent resources are removed.

---

## Stop

Stops a running resource without necessarily deleting it.

For example, stopping an EC2 instance does not automatically delete:

* Its EBS volume
* Its snapshots
* Some network resources
* Other associated services

Stopping does not always stop every charge.

---

## Terminate

Permanently removes an EC2 instance.

Termination is different from stopping.

---

## Enable or Disable

Changes whether a feature or setting is active.

Always read the scope and impact before confirming.

---

## Save Changes

Applies changes made on a configuration page.

Leaving a page without selecting Save may discard the changes.

---

## Cancel

Exits a workflow without applying the intended action.

Verify that no resource was created before assuming cancellation removed everything.

---

# 18. Common Beginner Mistakes

## Mistake 1: Creating Resources While Exploring

Opening a service is safe.

Selecting actions such as these can create resources:

```text
Create bucket
Launch instance
Create distribution
Create database
```

Do not continue through a creation wizard unless the lesson instructs you to do so.

---

## Mistake 2: Ignoring the Region Selector

A learner creates an EC2 instance in one Region and later looks for it in another.

Always record the Region used for each Regional resource.

---

## Mistake 3: Recreating a “Missing” Resource

Before recreating anything, check:

* Current Region
* Search filters
* Resource status
* Resource name
* Account identity
* Whether the original creation completed

Duplicate resources can create duplicate charges.

---

## Mistake 4: Treating the Root User as the Normal Administrator

The root user has unrestricted account access.

It should be reserved for limited account-level tasks that require root credentials. AWS explicitly recommends avoiding routine root-user access.

---

## Mistake 5: Creating Root Access Keys

Root access keys provide highly privileged programmatic access.

Do not create them for this manual.

The AWS CLI can be installed without root access keys.

---

## Mistake 6: Assuming the Console Saves Automatically

Many pages require you to select:

```text
Save
Update
Confirm
Create
Apply
```

After changing a setting, verify that the intended status is displayed.

---

## Mistake 7: Clicking Delete to Learn What It Does

Never use destructive actions as an experiment.

Some deletion operations cannot be reversed.

---

## Mistake 8: Assuming a Blank Dashboard Means No Charges

Billing data can be delayed.

Some dashboard widgets may not show all account information.

Check the Billing console directly.

---

## Mistake 9: Assuming a Budget Stops All Charges

A budget is mainly a tracking and notification tool.

You must still:

* Monitor services.
* Check bills.
* Delete resources.
* Protect the account.

---

## Mistake 10: Sharing Console Screenshots Without Review

Screenshots can expose:

* AWS Account ID
* Email address
* Billing information
* Resource names
* Public IP addresses
* Security settings
* Personal information

Crop and review every screenshot before publishing it.

---

## Mistake 11: Confusing the Account Menu with IAM

The account menu contains account-level and current-identity links.

IAM is the service used to manage users, groups, roles, policies, and permissions.

They are related but not the same.

---

## Mistake 12: Assuming All Services Are Regional

IAM and CloudFront do not behave like EC2.

Learn the scope of each service.

---

## Mistake 13: Changing Account Settings Without Recording Them

Before changing an important setting, record:

* Original value
* New value
* Reason
* Date
* Identity used

This becomes important in professional cloud administration.

---

## Mistake 14: Leaving the Root Session Open

When finished:

1. Open the account menu.
2. Select **Sign out**.
3. Close the browser tab.
4. Close the browser when using a shared device.

---

## Mistake 15: Using the Browser Back Button During Resource Creation

The browser Back button can produce confusing behaviour in multi-step AWS workflows.

Use the console’s:

* Cancel
* Previous
* Close
* Breadcrumb
* Service navigation

when available.

---

# 20. Console Troubleshooting

## Problem: The Console Does Not Load

Possible causes include:

* Internet interruption
* Browser problem
* Expired session
* Blocked cookies
* Browser extension interference
* AWS service issue

Try:

1. Refresh the page.
2. Confirm your internet connection.
3. Sign out and sign in again.
4. Use a supported, updated browser.
5. Temporarily test in a private browser window.
6. Disable unnecessary browser extensions.
7. Review AWS service-health information.

---

## Problem: You Are Repeatedly Signed Out

Possible causes include:

* Session expiration
* Browser cookie restrictions
* Network changes
* Multiple sign-in sessions
* Security controls
* Incorrect system time

Check:

* Browser cookies
* Computer date and time
* Internet stability
* Whether you are using a private browsing session

---

## Problem: A Service Does Not Appear in Search

Try:

* The full service name
* The common abbreviation
* The Services menu
* All services
* A related keyword

Examples:

```text
EC2
Elastic Compute Cloud

S3
Simple Storage Service

IAM
Identity and Access Management
```

Administrators can also control which Regions and services appear in console navigation, so hidden options can affect visibility in managed environments.

---

## Problem: A Resource List Is Empty

Before creating another resource, check:

1. Region selector
2. AWS account
3. Current identity
4. Search filters
5. Resource state
6. Resource type
7. Creation status

This is especially important for EC2, where resources are Region-specific.

---

## Problem: Billing Is Not Visible to an IAM User

Possible reasons include:

* IAM billing access has not been activated at the account level.
* The user lacks required billing permissions.
* The user is signed into the wrong account.
* The billing page is temporarily unavailable.

The account-level IAM billing-access setting must be activated before IAM policies can grant appropriate billing-console access.

---

## Problem: The Region Selector Is Missing or Disabled

Possible reasons include:

* The current service is global.
* The page is account-wide.
* The browser window is narrow.
* The navigation menu is collapsed.
* Account settings hide certain Regions.

IAM does not require Region selection in the normal way.

---

## Problem: The Account Menu Looks Different

The account menu changes according to:

* Root-user sign-in
* IAM-user sign-in
* Role session
* Federated sign-in
* Console update
* Screen size
* Permissions

Identify the currently signed-in identity before continuing.

---

## Problem: You Cannot Find Security Credentials

Try:

1. Open the upper-right account or identity menu.
2. Look for **Security credentials**.
3. Search for IAM if the identity-specific option is not available.
4. Confirm whether you are signed in as root, an IAM user, or a role.

AWS documents the Security credentials link under the upper-right account or identity menu.

---

## Problem: The Root Email Address Is Visible

Avoid capturing or sharing the screen.

Close any screen-sharing session and protect the account information.

---

# 21. Console Security Rules

```text
Who am I signed in as?
```

Possible answers:

* Root user
* IAM user
* IAM role
* Federated identity

---

## Rule 2: Check the Region

Before creating or changing a Regional resource, confirm:

```text
Which Region is selected?
```

---

## Rule 3: Check the Service

Confirm that you are in the correct service console.

For example:

```text
S3 bucket policy
```

must be configured in the relevant S3 bucket, not in EC2.

---

## Rule 4: Read the Entire Page

Do not only read the button text.

Review:

* Warnings
* Pricing notes
* Defaults
* Security implications
* Region
* Resource names
* Dependencies

---

## Rule 5: Do Not Create Credentials Without a Plan

Credentials must have:

* A defined owner
* A defined purpose
* Minimum required permissions
* Secure storage
* A removal or rotation plan

---

## Rule 6: Sign Out of Root

The root user is reserved for limited account-level tasks.

Once root security and IAM administration are configured, routine work must move to the IAM administrative identity.

**Answer:** No.

Billing data can be delayed, and widgets may show only summaries.

Does an AWS Budget automatically stop every AWS charge?

**Answer:** No.

A budget primarily tracks usage or cost and sends configured notifications.

What is a service console?

**Answer:** It is the part of the AWS Management Console used to manage one AWS service, such as Amazon S3, EC2, or IAM.

## Question 19

Does adding a service to favourites create a resource?

**Answer:** No.

## Question 20
