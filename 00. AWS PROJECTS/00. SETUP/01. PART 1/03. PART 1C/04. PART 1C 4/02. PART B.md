# Part 1C.4B – Creating the IAM Administrator

## Purpose of This Chapter

Part **1C.4A** introduced the concepts behind AWS Identity and Access Management (IAM), including users, groups, policies, authentication, authorization, and the principle of least privilege.

This chapter moves from theory to practice.

You will perform the one-time configuration required to create a dedicated administrative identity for everyday management of your AWS account.

From this point onward, you will begin transitioning away from using the AWS account root user for normal administration.

---

# Section 1 — Introduction

## Lesson Objective

In this lesson, you will:

* Create an **Administrators** user group.
* Attach the **AdministratorAccess** policy to the group.
* Create your first administrative IAM user.
* Configure AWS Management Console access.
* Add the new user to the Administrators group.
* Verify the configuration.
* Prepare to sign in as the new administrator in the next chapter.

By the end of this lesson, your AWS account will have a dedicated administrator that can be used for daily management instead of the root account.

---

## Prerequisites

Before continuing, confirm that you have completed:

* Part 1C.1 – Creating the AWS Account
* Part 1C.2 – Understanding the AWS Console
* Part 1C.3 – Securing the Root User
* Part 1C.4A – Understanding AWS IAM

Your root account should already have:

* A strong password
* Multi-factor authentication (MFA) enabled
* No root access keys
* Recovery information reviewed

---

## What You Will Build

During this lesson you will create the following structure:

```text
AWS Account
│
├── Root User
│
└── IAM
    │
    ├── User Group
    │      Administrators
    │
    └── User
           your-admin-user
```

The administrator user created in this chapter will become your primary account for managing AWS throughout the remainder of this handbook.

---

## One-Time Use of the Root User

For this lesson only, you should still be signed in as the **root user** because creating the first administrator is an account setup task.

Once the administrator has been created and verified, you will stop using the root account for normal daily administration.

In **Part 1C.4C**, you will sign out of the root account and begin using the newly created administrator account instead.

---

# Section 2 — Create the Administrators Group

## Step 1 – Open the IAM Console

1. Sign in to the AWS Management Console.
2. Confirm that you are signed in as the **root user**.
3. Use the search bar at the top of the console.
4. Search for:

```text
IAM
```

5. Select **IAM** from the search results.

The IAM Dashboard will open.

---

## Step 2 – Open User Groups

In the navigation pane on the left:

1. Select **User groups**.
2. Review the page.
3. Select **Create group**.

---

## Step 3 – Enter the Group Name

In the **User group name** field, enter:

```text
Administrators
```

Use the name exactly as shown for consistency throughout this handbook.

---

## Step 4 – Continue

Do not create the group yet.

You will attach the required permissions in the next section before completing the creation process.

---

## Step 5 – Review Before Creation

Before proceeding, verify:

* You are creating a **User group**.
* The group name is **Administrators**.
* No users have been added yet.
* No policy has been attached yet.

Everything should still be editable.

---

# Section 3 — Attach the AdministratorAccess Policy

> **Reminder:** In Part **1C.4A**, you learned that the `AdministratorAccess` AWS managed policy provides broad administrative permissions. In this section, you will attach that policy to the **Administrators** group.

---

## Step 1 – Search for the Policy

Within the Create Group page:

1. Locate the **Permissions** section.
2. Use the search box.
3. Search for:

```text
AdministratorAccess
```

AWS will display the matching policy.

---

## Step 2 – Select the Policy

Locate:

```text
AdministratorAccess
```

Select the checkbox next to the policy.

---

## Step 3 – Review the Summary

Confirm:

* The policy name is **AdministratorAccess**.
* The policy type is **AWS managed**.
* The policy is selected.

---

## Step 4 – Create the Group

Select **Create group**.

AWS will create the **Administrators** group.

---

## Step 5 – Verify the Attachment

Open the newly created **Administrators** group.

Confirm that:

* The group exists.
* The **Permissions** tab lists **AdministratorAccess**.
* No errors are displayed.

---

# Section 4 — Create the IAM Administrator User

## Step 1 – Open Users

From the IAM navigation pane:

1. Select **Users**.
2. Select **Create user**.

---

## Step 2 – Choose a Username

Enter a username.

Example:

```text
ann-admin
```

Use a clear and descriptive name.

Examples include:

```text
john-admin
mary-admin
company-admin01
```

Avoid vague names such as:

```text
user1
admin
test
newuser
```

---

## Step 3 – Continue

After entering the username, continue to the next configuration page.

Do not assign permissions yet.

That will be completed by selecting the **Administrators** group in a later step.

---

# Section 5 — Configure Console Access

## Step 1 – Enable Console Access

Enable AWS Management Console access for the new user.

This allows the user to sign in through the AWS web console.

---

## Step 2 – Create a Password

Create a strong password.

A good password should:

* Be unique.
* Be difficult to guess.
* Not be reused from another website.
* Be stored securely.

---

## Step 3 – Password Reset Option

If AWS presents the option to require a password reset at first sign-in:

* Enable it when creating accounts for other people.
* For a personal learning account, choose the option that best fits your workflow and security requirements.

---

## Step 4 – Review Settings

Before continuing, verify:

* Console access is enabled.
* The username is correct.
* The password has been configured.
* No warnings are displayed.

---

## Security Reminder

Do not:

* Share the password.
* Store it in plain text.
* Email it without protection.
* Reuse your AWS password elsewhere.

---

# Section 6 — Add the User to the Administrators Group

## Step 1 – Choose Existing Group

When AWS asks how permissions should be assigned:

Select:

**Add user to group**

---

## Step 2 – Select the Group

Choose:

```text
Administrators
```

The user will receive the permissions associated with this group.

---

## Step 3 – Continue

Verify that the correct group is selected.

Proceed to the review page.

---

# Section 7 — Review, Create, and Verify

## Step 1 – Review Configuration

Review:

* Username
* Console access
* Password settings
* Group membership

Ensure everything is correct before creating the user.

---

## Step 2 – Create the User

Select **Create user**.

AWS will provision the new administrator account.

---

## Step 3 – Save the Sign-In Information

AWS provides important sign-in details.

Save:

* The IAM sign-in URL.
* The username.
* Any temporary password if applicable.

Store this information securely.

---

## Step 4 – Download Credentials (If Offered)

If AWS provides a downloadable credentials file:

* Download it.
* Store it securely.
* Avoid leaving it in public folders.

---

## Step 5 – Verify the User

Return to the **Users** page.

Confirm that the new user appears in the list.

---

## Step 6 – Verify Group Membership

Open the user.

Confirm that:

```text
Administrators
```

appears under **Groups**.

---

## Step 7 – Verify Permissions

Open the **Permissions** tab.

Confirm that the user receives administrative permissions through the **Administrators** group.

---
